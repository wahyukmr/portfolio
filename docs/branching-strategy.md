# Branching Strategy

## Main Branches
- main: Production-ready code
- develop: Integration branch

## Supporting Branches
- feature/*: New features
- fix/*: Bug fixes
- chore/*: Maintenance
- refactor/*: Code restructuring

## Workflow
1. Create issue
2. Create branch from develop
3. Commit changes
4. Open Pull Request
5. Merge into develop
6. Periodically merge develop into main
